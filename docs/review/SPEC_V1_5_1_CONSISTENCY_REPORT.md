# SPEC v1.5.1 Consistency Report

作成日: 2026-06-25

対象文書:

- `docs/md/spec_v1.5.md`
- `docs/md/spec_v1.5.1.md`

本レポートは、v1.5で第19章に統合された承認済み現行仕様と、第1〜18章に残っていた旧仕様の矛盾を解消した結果を記録する。コード、Migration、Docker、DB、設定ファイル、依存関係、既存データは変更していない。

## 意図的に残した記述

- 「個別確率」は、ユーザー画面で表示しないことを明記するために残した。
- 公開APIの景品別`ppm`は、現行コードが返却しているため、仕様として残した。ただしUI表示対象外である。
- 付録Cの`完全ハズレ`、`no_prize`、`期限値`等は、過去版からの変更履歴説明として残した。現行仕様としては採用しない。
- `point_lots`の有効期限という語は、freeポイントの期限管理とロット構造の説明として残した。paidは`expire_at = null`で期限なしと明記している。

## 未解決の仕様判断

1. 公開APIから景品別`ppm`を将来除外するかどうか。
   - 現行コード: `backend/app/Http/Resources/GachaDetailResource.php` が景品ごとの`ppm`を返却している。
   - v1.5.1判断: 既存API互換維持のため、UIのみ非表示、公開APIには含める。

2. Googleログインのユーザー側フロント導線の完成範囲。
   - Laravel API基盤は実装済み。
   - ユーザー側画面の導線は、別担当作業との整合確認が必要。

3. 最低保証枠の額・内容。
   - 仕様として最低保証ありは確定。
   - 額、還元率、法務上の妥当性は公開前確認事項。

4. 送料、配送運用、同梱可否。
   - 内部管理は景品単位。
   - 実運用で送料をどう扱うかは規約・FAQ・画面文言と合わせて確定が必要。

## RELEASE BLOCKER

1. 本番決済プロバイダ接続、Webhook署名検証、返金、チャージバック、3Dセキュア等の実装。
2. mock決済が本番環境で誤利用されない制御の最終確認。
3. 日次ポイント残高スナップショット作成処理。
4. 基準日残高集計、管理表示、CSV出力。
5. 返金・チャージバック時のポイント取消処理。
6. 残高不足時のマイナス残高又は利用制限処理。
7. 資金決済法、供託、未使用残高、サービス終了時払戻し、規約文言の法務・会計確認。

## 売上管理

売上管理機能はv1.5.1でも未実装のままとする。v1.6予定機能として、月別・日別売上、決済種別別売上、無償/有償ポイント消費、ポイント消費詳細の設計を別途行う。

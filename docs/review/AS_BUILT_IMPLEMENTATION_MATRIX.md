# AS-BUILT IMPLEMENTATION MATRIX

作成日: 2026-06-25

対象:

- 正本: `docs/md/spec_v1.4.md`
- 現行正本: `docs/md/spec_v1.5.md`

- 承認決定: `docs/decisions/APPROVED_AS_BUILT_SPECIFICATIONS_2026-06-25.md`

監査方針:

- `docs/md/all_check.md` の記載だけでは実装済み判定しない。
- コード、Migration、API、画面、テストファイルを確認した範囲で判定する。
- 本監査では、ユーザー指示によりコード、DB、Docker、設定ファイルを変更していない。
- 本監査では、`php artisan migrate`、Docker build、依存関係更新、テスト実行を行っていない。
- 「テスト実行結果」は、既存の作業ログ `worklogs/codex-main.md` に記録された過去実行結果、または「本監査では未実行」として記載する。

ステータス定義:

- `VERIFIED`: 正式仕様として承認済み。コードとテストファイル、または作業ログ上のテスト実行結果で確認済み。
- `IMPLEMENTED_UNTESTED`: 正式仕様として承認済み。コードは確認できるが、自動テスト、E2E、実機確認が不足している。
- `PARTIAL`: 現在実装されている挙動は正式承認済み。未実装部分は残課題として維持する。
- `NOT_FOUND`: リポジトリまたは確認対象環境で実装を確認できていない。実装済みとは断定しない。
- `PENDING`: 未実装。正式実装としては未承認で、バックログとして扱う。
- `FUTURE`: 将来対応。現在の実装範囲外。

## Approval Handling

2026-06-25に、プロジェクト責任者は `docs/md/all_check.md` に記載され、本マトリクスで `VERIFIED`、`IMPLEMENTED_UNTESTED`、`PARTIAL` と確認された現行実装を正式仕様として承認した。

今後、これらを「人間の仕様承認待ち」「正式採用するか判断が必要」とは扱わない。ただし、承認状況と実装・テスト状況は分けて管理する。

- `VERIFIED`: 仕様承認済み、実装確認済み、テスト根拠あり。
- `IMPLEMENTED_UNTESTED`: 仕様承認済み、実装確認済み、テスト又はE2E確認不足。
- `PARTIAL`: 実装済み部分は仕様承認済み。未実装部分は残課題。
- `PENDING`、`FUTURE`、`NOT_FOUND`: 承認済み実装には含めない。

mock決済は開発・検証用機能として承認済みだが、本番決済機能ではない。本番決済プロバイダ接続は `PENDING` として扱う。
